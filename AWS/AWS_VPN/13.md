# PART 13 - Deploy Pritunl VPN Server for Secure Access to Private EC2

---

# Why Do We Need a VPN?

Our EC2 instance is deployed inside a **private subnet**.

It has:

* No Public IP
* No Internet Gateway Route
* No direct SSH access from the Internet

Instead of exposing the server publicly, we deploy a VPN server.

After connecting to the VPN, our laptop becomes part of the AWS private network and can directly access private resources using their private IP addresses.

Example:

```
Laptop

↓

Pritunl VPN Tunnel

↓

Public Pritunl Server

↓

AWS VPC

↓

Private EC2

10.10.10.10
```

This is a common enterprise architecture for secure remote administration.

---

# What is Pritunl?

Pritunl is an open-source VPN server built on OpenVPN and WireGuard.

Features:

* Free and Open Source
* Web-based Management UI
* Multi-user Support
* SSL Encryption
* Supports OpenVPN and WireGuard Clients
* MFA Support
* Easy Certificate Management

# Step 13.7 - SSH into VPN Server

```bash
ssh -i mykey.pem ec2-user@<Elastic-IP>
```

Update packages

```bash
sudo dnf update -y
```

---

# Step 13.8 - Install Docker

```bash
sudo dnf install docker -y

sudo systemctl enable docker

sudo systemctl start docker

sudo usermod -aG docker ec2-user
```

Reconnect to the server.

Verify

```bash
docker --version
```

---

# Step 13.9 - Deploy MongoDB

```bash
docker run -d \
--name mongodb \
--restart always \
-p 27017:27017 \
mongo:latest
```

Verify

```bash
docker ps
```

---

# Step 13.10 - Deploy Pritunl Server

```bash
docker run -d \
--name pritunl \
--restart always \
--network host \
-e MONGO_URI=mongodb://127.0.0.1:27017/pritunl \
pritunl/pritunl
```

Verify

```bash
docker ps
```

# Step 13.19 - Connect to Private EC2 via Private IP

Verify connectivity

```bash

```

```bash
ssh -i mykey.pem ec2-user@10.10.10.10
```

You are now accessing the EC2 instance directly over the private network without exposing it to the Internet.

# Production Benefits

* No public IP required on private EC2 instances
* Encrypted VPN tunnel for administration
* Direct access using private IP addresses
* Centralized remote access management
* Scales to multiple administrators and servers
* Aligns with enterprise security best practices by reducing the attack surface
